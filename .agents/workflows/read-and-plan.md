# Read and Plan Workflow

Use this workflow at the start of every Antigravity session.

## Files to Read First
1. `.agents/project-brief.md`
2. `.agents/implementation-rules.md`
3. `.agents/current-state.md`
4. `.agents/session-log.md`
5. The active task file inside `.agents/tasks/`

## Required Behavior
- Treat these files as the source of truth for the session
- Summarize the active task in your own words
- Identify the exact files likely to change
- Separate:
  - what is already true in the repo
  - what still needs to be built
- Propose a step-by-step implementation plan
- Do not implement anything yet
- Wait for explicit approval before execution
- Do not commit
- Do not push

## Expected Output Format
1. Task understanding
2. Files likely to change
3. Implementation plan
4. Risks / edge cases
5. First recommended step only
