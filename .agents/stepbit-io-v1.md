# Stepbit I/O Contract — Version 1.0

## Purpose
This document defines the formal communication contract between **Stepbit-core** (Orchestrator) and **QuantLab** (Research Engine). 

QuantLab remains research-first and CLI-driven. This contract provides a stable wrapper for machine-to-machine interactions.

---

## 1. Request Schema (Input)

Stepbit invokes QuantLab via CLI. For complex parameter sets, QuantLab reads a JSON request block.

### JSON Fields
| Field | Type | Description |
|---|---|---|
| `schema_version` | `string` | Must be `"1.0"` |
| `request_id` | `string` | Unique ID generated by Stepbit for tracking. |
| `command` | `string` | One of: `run`, `sweep`, `forward`, `portfolio`. |
| `params` | `dict` | Command-specific parameters. |

### Example: Strategy Backtest Request
```json
{
  "schema_version": "1.0",
  "request_id": "req_550e8400",
  "command": "run",
  "params": {
    "ticker": "ETH-USD",
    "strategy": "RsiMaAtrStrategy",
    "strategy_params": {
      "rsi_buy_max": 30.0,
      "rsi_sell_min": 70.0
    },
    "start_date": "2023-01-01",
    "end_date": "2023-12-31"
  }
}
```

---

## 2. Response Schema (Output)

QuantLab produces a standard JSON envelope upon completion (success or failure).

### Success Response
```json
{
  "schema_version": "1.0",
  "request_id": "req_550e8400",
  "status": "success",
  "run_id": "run_20260319_221500_deadbeef",
  "fingerprint": "f6e8b2abc123...",
  "artifacts_path": "./outputs/runs/run_20260319_221500_deadbeef",
  "metrics": {
    "sharpe": 1.82,
    "total_return": 0.45,
    "max_drawdown": -0.15,
    "win_rate": 0.62
  }
}
```

### Failure Response
```json
{
  "schema_version": "1.0",
  "request_id": "req_550e8400",
  "status": "error",
  "error": {
    "code": "DATA_ACCESS_ERROR",
    "message": "OHLC data missing for range 2023-01-01 -> 2023-12-31",
    "details": "YFinance returned empty DataFrame."
  }
}
```

---

## 3. Identity & Fingerprinting

| Identity | Type | Purpose |
|---|---|---|
| **run_id** | Temporal | Unique per execution (includes timestamp). Used for filesystem storage. |
| **fingerprint** | Deterministic | Hash of `(command, params)`. Used for caching and detecting redundant research. |

---

## 4. Exit Codes

QuantLab will align with standard exit codes for machine readability:

| Code | Label | Meaning |
|---|---|---|
| `0` | `SUCCESS` | Task completed normally. |
| `1` | `GENERAL_ERROR` | Unexpected crash or logic error. |
| `2` | `INVALID_CONFIG` | Request JSON or CLI flags are invalid. |
| `3` | `DATA_ERROR` | Required data not found or corrupted. |
| `4` | `STRATEGY_ERROR` | Strategy execution error. |

---

## 5. Artifact Paths
Outputs are always stored in:
`outputs/runs/<run_id>/`

Mandatory artifacts:
- `metadata.json`: Identity and context.
- `config.json`: The exact parameters used.
- `report.json`: Metrics and summary.
