

# Task: Define roadmap after CLI refactor

## Goal
Decide the next major development direction for QuantLab after CLI modularization is stabilized.

## Why
Without an explicit post-refactor roadmap, the project risks mixing internal stabilization, research improvements, portfolio work, and external integration without a clear sequence.

## Scope
- assess the current state of QuantLab after the CLI refactor
- identify the next logical development block
- compare candidate directions such as:
  - run registry tooling
  - strategy research tooling
  - portfolio analysis improvements
  - experiment automation
  - Stepbit integration milestone 1
- produce a clear recommendation

## Non-goals
- implementing the chosen roadmap item
- creating broad product strategy beyond the next block
- forcing Stepbit integration prematurely

## Inputs
- current project state
- internal architecture status
- current roadmap notes
- issue #14 or equivalent

## Expected outputs
- short roadmap decision
- rationale for why that block comes next
- sequencing note for what is explicitly postponed
- cleaner prioritization across internal vs integration work

## Acceptance criteria
- the next development block is explicitly chosen
- the choice is justified against QuantLab’s identity
- postponed directions are named clearly
- the decision helps avoid cross-milestone confusion

## Constraints
- preserve QuantLab as a research-first, rigorous system
- prefer solidity over breadth
- do not let external integration eclipse internal coherence
## GitHub issue
- #14 Define roadmap after CLI refactor

## Suggested next step
Summarize QuantLab’s current maturity and choose the single most important next block before starting new implementation work.
