# Issue #210 — Typography, Spacing, and Panel Polish Pass

## Goal
Apply a final polish pass across typography, spacing, and panel treatment so the desktop feels more cohesive and production-ready.

## Why this matters
After the workstation restructure and hardening, the remaining gaps are now mostly compositional: rhythm, spacing, panel balance, and type hierarchy.

## Scope

### In scope
- typography rhythm
- spacing consistency
- panel edge treatment
- card/panel coherence

### Out of scope
- major layout changes
- new product surfaces
- runtime or data logic

## Relevant files

- `desktop/renderer/styles.css`
- `desktop/renderer/index.html`
- `desktop/renderer/modules/tab-renderers.js`

## Expected deliverable

A cleaner and more cohesive desktop UI that feels intentionally composed rather than incrementally accumulated.

## Done when

- spacing feels consistent across the shell
- type hierarchy is easier to read
- panels feel related parts of one system

## Notes

This issue should come after the more structural visual maturity slices above.
