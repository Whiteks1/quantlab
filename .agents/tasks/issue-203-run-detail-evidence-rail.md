# Issue #203 — Run Detail Evidence Rail

## Goal
Refine run detail so the right-side context becomes a stable evidence rail for artifacts, continuity, validation state, and operational links.

## Why this matters
Run detail is where QuantLab proves that a run is traceable, reviewable, and operationally grounded.

## Scope

### In scope
- right-rail evidence hierarchy
- artifact availability emphasis
- launch continuity visibility
- validation and decision state clarity

### Out of scope
- new backend fields
- compare redesign
- paper execution redesign

## Relevant files

- `desktop/renderer/modules/tab-renderers.js`
- `desktop/renderer/styles.css`

## Expected deliverable

A run detail view that feels like an evidence review surface rather than a generic detail card stack.

## Done when

- artifact and continuity state are visually easy to scan
- the evidence rail has clear hierarchy
- decision state remains visible without displacing artifacts
