# Desktop Visual Maturity — Proposed Issue Block

This issue block defines the next desktop UI slices needed to move QuantLab Desktop from a structurally correct workstation into a more mature and legible analytical product surface.

## Proposed sequence

1. Issue #206 — Desktop topbar and global chrome maturity
2. Issue #207 — Runs table visual semantics and density polish
3. Issue #208 — Right rail evidence panel maturity
4. Issue #209 — Status language and state signaling refinement
5. Issue #210 — Typography, spacing, and panel polish pass

## Intent

The goal is not to copy a trading dashboard. The goal is to reach a stronger level of visual maturity in the desktop while preserving QuantLab semantics:

- denser
- calmer
- more legible at a glance
- more review-oriented
- more workstation-like

## Rules for this block

- Keep QuantLab vocabulary: runs, artifacts, evidence, validation, continuity.
- Do not introduce portfolio-monitoring semantics as the primary language.
- Favor visible maturity gains over decorative effects.
- Keep the existing workstation hierarchy intact.
- Avoid changing backend contracts or runtime behavior in this block.
