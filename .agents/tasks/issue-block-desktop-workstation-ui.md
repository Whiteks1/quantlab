# Desktop Workstation UI — Proposed Issue Block

This issue block defines the next desktop UI slices needed to move QuantLab from a shell-like surface toward a denser workstation for research, artifacts, and supervised decisions.

## Proposed sequence

1. Issue #201 — Desktop workstation layout foundation
2. Issue #202 — Runs as primary work surface
3. Issue #203 — Run detail evidence rail
4. Issue #204 — Compare and candidates workbench density
5. Issue #205 — Assistant and command surface demotion

## Intent

The goal is not to imitate a trading dashboard. The goal is to make QuantLab Desktop feel like a native analytical workstation:

- denser
- more hierarchical
- more review-oriented
- less shell-first
- less demo-like

## Rules for this block

- Keep QuantLab semantics: runs, artifacts, evidence, validation, continuity.
- Do not shift the product toward portfolio-monitoring or generic trading UI language.
- Favor incremental slices over a broad rewrite.
- Remove replaced surfaces instead of keeping parallel versions.
