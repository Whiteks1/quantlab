# QuantLab v0.1 Release Evidence

This document serves as the canonical record of the visual and operational state for the **QuantLab v0.1 (April 2026)** milestone.

## Primary Work Surfaces (High Stability)

The following four surfaces represent the "Must-Ship" stable core of the v0.1 desktop shell.

### 1. System Dashboard (Telemetry)
- **Purpose**: Verify local runtime health, index size, and bridge connectivity.
- **Evidence Signal**: "Managed Server" status, "QuantLab shell running in local-only fallback mode" (when applicable), and Stepbit boundary status.

### 2. Runs Index (Workstation)
- **Purpose**: survey historically executed runs from the registry.
- **Evidence Signal**: Full table of indexed runs with metadata (Ticker, Mode, Created), "Candidate"/ "Shortlist" status indicators, and stable vertical scroll.

### 3. Artifact Explorer (Handoff)
- **Purpose**: Inspect the physical files produced by a quantitative run.
- **Evidence Signal**: Structured listing of canonical files (`report.json`, `config.json`, `metrics.json`) and the ability to open them directly from the shell.

### 4. Run Detail (Inspection)
- **Purpose**: Deep-dive into a specific run's results without browser dependency.
- **Evidence Signal**: Rich summary cards (Return, Sharpe, Drawdown), config provenance list, and decision state controls (Set Baseline, Add Shortlist).

---

## Secondary Features (Operational Support)

### Decision Compare
- **Status**: Functional and stabilized against text overflows.
- **Usage**: Used for ranking shortlisted candidates. While visually dense, it successfully performs cross-run metric comparison and config delta inspection.

### Paper Ops
- **Status**: Monitoring only.
- **Usage**: surfacing paper session health and broker boundary alerts.

---

## Visual Consistency Standards

1. **Hardened Layout**: Zero vertical stretching in the sidebar after initial hydration.
2. **Text Stability**: Summary cards utilize ellipsis for long IDs to maintain grid integrity.
3. **Graceful Fallbacks**: Missing data renders as structured empty states, not generic errors.
