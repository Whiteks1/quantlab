# QuantLab v0.1 Scope

**QuantLab Research v0.1 is a local-first workstation for launching, inspecting, and reviewing real runs through a stable desktop shell.**

This document outlines the explicit boundaries, supported flows, and known limitations of the v0.1 release milestone.

## What is in scope
- **Native Desktop Workstation**: A stable, unified shell unifying the workflow (`System`, `Runs`, `Paper Ops`).
- **Graceful Run Inspection**: The ability to inspect local runs, view results, read config deltas, and review canonical artifacts natively.
- **Decision Compare**: Ranking capability for runs, provided they have fully formed canonical metric outputs in the workspace registry.
- **Fallback Integrity**: Incomplete, legacy, or disconnected runs natively degrade by explaining what is locally available (metrics vs workspace directories) rather than rendering raw JS errors.

## What is out of scope
- **Live Autonomous Routing**: Fully autonomous agent-driven production deployment to varied CEX APIs.
- **Deep Web-based Shell Execution**: Pushing the legacy browser-based React components (`/launch`) as the canonical workflow.
- **Multi-user Sync**: Remote synchronized execution outside the local machine's `outputs` index boundary.

## Supported Happy Path
The primary narrative for v0.1 revolves around locally managed research and assessment:
1. **Boot**: `npm start` in the `desktop/` layer.
2. **System Setup**: Observing telemetry in the `System` strip (verifying `Mode local-only`, `QuantLab up`).
3. **Runs Indexing**: Surveying past runs via the central `Runs` view.
4. **Decision Validation**: Opening an individual Run to read its explicit metrics, provenance, workspace outputs, and canonical `.json` report.
5. **Comparison**: Using the `Compare` table for selecting ranked candidate runs sharing canonical output sets.

## Known Limitations
- **Strict Compare Matrix**: Cross-run ranking strictly requires that every run possesses canonical metrics (Return, Sharpe, Drawdown). Runs missing these are rejected from rendering in the rank table, defaulting gracefully to an informative empty state.
- **Artifact Shape Variance**: Runs from older workflow epochs or failed environments that lack a canonical manifest will render a structured fallback state, presenting the remaining accessible telemetry instead of throwing unstructured errors.
- **Legacy Launch**: Internal shell run initiation currently demotes the raw browser-framed launch tab to a "legacy fallback" accessible via the command palette, discouraging its formal use for the final native milestone.
